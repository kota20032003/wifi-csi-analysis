# Wi-Fi CSI Analysis

## 概要

卒業研究で使用したWi-Fi CSI解析用のPythonコードです．
ESP32から取得したCSIデータを用いて，CSIの取得，3D可視化，機械学習による分類，予測結果の後処理を行います．

## 背景

Wi-Fi CSI (Channel State Information: チャネル状態情報) は，Wi-Fi信号が空間を伝わるときの変化を表す情報です．
本研究では，ESP32で取得したCSIデータを用いて，環境変化や状態変化を観測することを目的としました．

## 主な機能

- ESP32からCSIデータを取得し，CSVに保存
- CSIの振幅をリアルタイムに可視化
- 時間，周波数，振幅の3Dグラフを表示
- IQ特徴量と振幅特徴量を用いたロジスティック回帰
- 状態遷移ルールによる予測ラベルのスムージング
- 多数決による予測ラベルのスムージング

## 各ファイルの説明

plot_csi_ravel2_old.py

ESP32からシリアル通信でCSIデータを取得し，CSVに保存するコードです．
取得したCSIデータのうち，指定したサブキャリアのI/Q成分から振幅を計算し，リアルタイムでグラフ表示します．
また，取得時刻をマイクロ秒単位で記録することで，後から時系列データとして解析しやすい形にしています．

plot_3d_freq.py

取得したCSIデータを時間，周波数，振幅の3次元グラフとして可視化するコードです．
各サブキャリアのI/Q成分から振幅を計算し，時間方向に並べることで，CSIの変化を視覚的に確認できるようにしています．
また，周波数軸を作成し，サブキャリアごとの変化を周波数方向から確認できるようにしています．

csi_experiment.py

CSIデータを用いてロジスティック回帰による分類実験を行うコードです．
csi_payloadからI/Qペアを取り出し，ガード帯とDC成分を除外したうえで，IQ特徴量と振幅特徴量を作成します．
その後，クラス重みの有無を変えた4パターンで学習と評価を行い，予測結果をCSVとして保存します．

smooth_fsm.py

ロジスティック回帰で得られた予測ラベルに対して，状態遷移ルールを用いた後処理を行うコードです．
あらかじめ定めた状態遷移に反する予測が出た場合は，前の状態を維持することで，不自然なラベル変化を抑えます．

smooth_majority.py

予測ラベルに対して，多数決によるスムージングを行うコードです．
一定幅の窓の中で最も多く出現したラベルを採用することで，瞬間的な誤分類や細かい揺れを抑えます．

## 主な関数の説明
plot_csi_ravel2_old.py
ts_iso_us_slash()

現在時刻をマイクロ秒単位の文字列に変換する関数です．
CSIデータをCSVに保存するとき，取得時刻を細かく記録するために使用します．


csi_data_handler()

シリアルポートからCSIデータを読み取り，CSIペイロードを抽出し，CSV保存とグラフ表示用データの更新を行う関数です．
受信した文字列からCSI_DATAを含む行だけを対象にし，末尾の角括弧内にあるI/Qデータを取り出します．


plot_3d_freq.py
build_freq_axis_mhz()

通信帯域幅とFFTサイズから，サブキャリアに対応する周波数軸を作成する関数です．
3Dグラフで周波数方向の位置を表示するために使用します．


payload_to_amplitudes()

csi_payloadの文字列を数値列に変換し，I/Q成分から振幅を計算する関数です．
データ数が不足している場合は0で補い，多い場合は必要な長さに切り詰めます．


detect_wall_bin_from_train()

train.csvを読み込み，各サブキャリアの平均振幅を調べ，突出して大きいビンを検出する関数です．
特定のビンが過度に大きい場合，グラフ表示への影響を抑えるために壁ビン候補として扱います．


csi_data_handler()

シリアル通信でCSIデータを取得し，I/Q成分から振幅を計算して，3D表示用の履歴データに追加する関数です．


setup_big_window()

Matplotlibの表示ウィンドウをできるだけ大きく表示するための関数です．
OSや描画環境によって処理が異なるため，失敗しても処理を継続するようにしています．


main()

CSI取得用スレッドを開始し，3Dグラフを更新し続けるメイン処理です．
時間，周波数，振幅の3軸でCSIの変化を表示します．


csi_experiment.py
parse_iq_pairs()

csi_payloadの文字列を，I/Q成分のペアに変換する関数です．
例えば，スペース区切りの数値列を，[(I1，Q1)，(I2，Q2)，...] の形に変換します．


make_features_iq()

I/Q成分をそのまま特徴量として使うための関数です．
ガード帯やDC成分など，データを載せないサブキャリアを除外したうえで，I/Q値を1次元の特徴量ベクトルに変換します．


make_features_amp()

I/Q成分から振幅を計算し，振幅特徴量を作成する関数です．
振幅は sqrt(I^2 + Q^2) で計算します．


run_experiment()

1つの条件について，ロジスティック回帰の学習，予測，評価，CSV保存をまとめて行う関数です．
正解率，適合率，再現率，F値，混同行列を出力します．


main()

学習データとテストデータを読み込み，IQ特徴量と振幅特徴量を作成し，4パターンの分類実験を実行するメイン処理です．


smooth_fsm.py
next_allowed_states()

現在の状態から次に遷移してよい状態の集合を返す関数です．
状態遷移のルールを定義する役割を持ちます．


smooth_with_fsm()

予測ラベル列に対して，状態遷移ルールに基づく補正を行う関数です．
許可されていない遷移が出た場合は，前の状態を維持します．


main()

予測結果CSVを読み込み，状態遷移ルールによるスムージングを行い，補正後のCSVを保存するメイン処理です．


smooth_majority.py
smooth_with_majority()

指定した窓幅の中で多数決を行い，予測ラベルをなめらかにする関数です．
同数になった場合は元のラベルを優先し，それでも決まらない場合は最小のクラス番号を採用します．


main()

予測結果CSVを読み込み，多数決スムージングを行い，補正後のCSVを保存するメイン処理です．




## 工夫した点
1．CSIデータを解析しやすい形に変換した点

ESP32から取得したCSIデータは，I/Q成分がスペース区切りで並んだ文字列として出力されます．
そのままでは機械学習や可視化に使いにくいため，I/Qペアへの変換，振幅計算，特徴量ベクトル化を行いました．

2．ガード帯とDC成分を除外した点

CSIデータのうち，データを載せないサブキャリアやDC成分は，分類に有効な情報を持ちにくいと考えられます．
そのため，これらの成分を除外してから特徴量を作成することで，解析に使う情報を整理しました．

3．複数の特徴量で比較できるようにした点

I/Q成分をそのまま使う場合と，振幅に変換して使う場合の両方を実装しました．
これにより，どの特徴量が分類に有効かを比較できるようにしました．

4．クラス不均衡への対応を試した点

ロジスティック回帰では，クラス重みなしの場合と，クラス重みありの場合の両方を実行しました．
データ数に偏りがある場合でも，分類結果への影響を確認できるようにしています．

5．評価指標を複数出力した点

正解率だけでなく，適合率，再現率，F値，混同行列を出力するようにしました．
これにより，単に正解した割合だけでなく，どのクラスを誤分類しやすいかも確認できます．

6．リアルタイム可視化を行えるようにした点

CSIを取得しながら，指定サブキャリアの振幅や3Dグラフをリアルタイムに表示できるようにしました．
これにより，データ取得中に信号の変化を確認できます．

7．予測結果の後処理を実装した点

機械学習の予測結果には，瞬間的な誤分類や不自然なラベル変化が含まれる場合があります．
そのため，状態遷移ルールと多数決スムージングの2種類の後処理を実装し，時系列として自然な結果に近づける工夫をしました．

8．設定値をコード上部にまとめた点

COMポート番号，ボーレート，窓幅，出力ファイル名などをコード上部にまとめました．
これにより，実験環境が変わった場合でも，必要な箇所だけを変更して実行しやすくしています．

9．異常な入力や不足データに対応した点

CSIデータの形式が崩れている場合や，データ数が不足している場合でも，処理が止まりにくいようにしています．
データ取得中に一部の行が不完全でも，可能な範囲で処理を継続できるようにしました．
